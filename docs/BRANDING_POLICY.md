# FridaMusic TM Branding Policy

## Purpose
This policy protects the FridaMusic TM and Frida Labs TM brand identity while keeping source code collaboration open under Apache License 2.0.

## Allowed uses
You may:
- Mention FridaMusic TM and Frida Labs TM in documentation, comparisons, and technical references.
- Contribute code, fixes, and features through issues and pull requests.
- Create educational or experimental forks under Apache 2.0.
- Refer to the original project as the upstream source.

## Not allowed without explicit authorization
You may not:
- Use official FridaMusic TM or Frida Labs TM logos, icons, or artwork in redistributed apps.
- Use official project names as primary app branding for derivative builds.
- Publish APK/AAB binaries that use official visual identity or imply official status.
- Clone or reproduce the full official visual identity in a confusing way.
- Present derivative projects as the official FridaMusic TM app.

## Fork naming guidance
Forks should use clearly differentiated names, package IDs, app icons, and store listings to avoid confusion with official releases.

## Enforcement principle
The project aims for collaborative, pro-open-source enforcement: protect users from confusion while preserving legal and technical freedom to fork source code under Apache 2.0.
